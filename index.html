Documentation Cryptographique

1. Introduction

Dans un monde où les données sont sensibles, assurer leur sécurité est primordial. Cette application utilise les concepts fondamentaux de la cryptographie moderne.

2. Principe de Kerckhoffs

"Un système cryptographique doit être sécurisé même si tout est connu, sauf la clé secrète."
👉 La sécurité repose uniquement sur la clé et non sur l'algorithme.

3. Échange de clés Diffie-Hellman

Permet d'échanger une clé secrète sur un canal non sécurisé. Cependant, il est vulnérable à l'attaque de l'homme du milieu (MITM). La solution est l'utilisation de TLS et de certificats numériques pour authentifier les parties.

4. Chiffrement Asymétrique (RSA)

Utilisation de deux clés :

  • Clé publique → Chiffrement
  • Clé privée → Déchiffrement

Avantages : Sécurité, distribution facile des clés.
Limites : Lent → souvent combiné avec AES (chiffrement hybride).

5. Signature Numérique

Utilise un hachage (SHA-256) pour garantir l'intégrité et l'authenticité :

  • Signature avec la clé privée
  • Vérification avec la clé publique

6. Certificat X.509

Le rôle de l'Autorité de Certification (CA) est d'émettre des certificats qui lient une identité à une clé publique. La chaîne de confiance permet de vérifier l'authenticité. Limites : certificats auto-signés, attaques sur les CA.

Security First - Cette application respecte le principe de Kerckhoffs et ne stocke aucune clé privée.
